Privacy Policy

Fokus — Minimalist Launcher  ·  com.himanshoe.fokus

Developer: Himanshu  ·  Last updated: June 2025

1. Overview

Fokus ("the App") is an Android launcher that provides app time limits, Focus Shield (distraction blocking), Health Connect integration, on-device AI summaries, and a notification panel. This Privacy Policy explains what data the App accesses, how it is used, and what is never collected.

The App is designed with privacy as a core principle. Most features work entirely on-device. No personal data is sold or shared with third parties for advertising.

2. Data the App Accesses

2.1 App Usage Statistics

Permission: PACKAGE_USAGE_STATS

Android's UsageStats API provides aggregate screen-time data — which apps were used, for how long, and on which dates. This powers the Stats screen and Focus Shield time-limit enforcement.

Raw usage-stats data is never uploaded to any server. It is processed on-device and stored locally in a Room database. Data is cleared when the app is uninstalled.

2.2 Installed Apps

Permission: QUERY_ALL_PACKAGES

A list of all installed apps is required to populate the launcher home screen grid and let users configure Focus Shield rules. This data is held locally and never uploaded.

2.3 Health Connect Data

Fokus optionally reads from Health Connect to display a daily health summary. The following data types are read only if the user grants permission:

It is processed exclusively on-device — including by the optional on-device AI summary powered by the Gemma 3 model running locally. Users can revoke Health Connect permissions at any time via the Health Connect app.

2.4 Accessibility Service

Permission: BIND_ACCESSIBILITY_SERVICE

Fokus registers an Accessibility Service exclusively for Focus Shield. Specifically:

  • Scroll limit enforcement — detects scroll events so the app can overlay Focus Shield when the configured limit is exceeded.
  • Feature blocking — detects when a blocked feature (Shorts, Reels, Search, Comments) becomes visible and overlays Focus Shield before the content is viewed.
  • Daily time limit enforcement — detects when a time-limited app comes to the foreground after its daily limit has been reached.

The Accessibility Service does not read, record, store, or transmit any on-screen text, images, passwords, messages, or personal information. Window content is inspected only to identify which feature is active; the content itself is discarded immediately. The service is not used for analytics, advertising, or any purpose beyond the three Focus Shield functions described above.

2.5 Notification Listener

Permission: BIND_NOTIFICATION_LISTENER_SERVICE

Fokus registers a Notification Listener to power the in-app notification panel. Notification content (message body, sender name, text) is never logged, stored, or transmitted. Only the app package name is used for display. Notification data is processed in memory only and discarded when the notification is dismissed.

2.6 Calendar

Permission: READ_CALENDAR

Calendar events are optionally displayed in a home screen widget. Events are read from the device's CalendarProvider and displayed in the widget. They are not stored in the App's database and are never uploaded.

2.7 Location

Permission: ACCESS_COARSE_LOCATION

Coarse (city-level) location is used solely to fetch current weather conditions for the optional weather widget. The location is passed to one weather API request and is not stored. Precise location is never requested.

2.8 Analytics — Firebase Analytics

What is collected: anonymised event names (e.g. screen views, feature interactions), whether a search returned results (boolean — query text is never logged), premium feature interactions.

What is never collected: no names, emails, or PII; no notification content; no on-screen text; no Health Connect values; no search query text.

Firebase Analytics automatically anonymises IP addresses. Processed by Google LLC under their Privacy Policy.

2.9 Crash Reporting — Firebase Crashlytics

Crashlytics collects anonymised crash reports (stack traces, device model, OS version) to help diagnose bugs. No personal data is included in crash reports.

2.10 Billing — RevenueCat

Purchases are processed by Google Play Billing. RevenueCat receives an anonymised customer identifier and purchase record to manage Pro entitlements. No payment card data is accessible to the App or to RevenueCat. See RevenueCat's Privacy Policy.

3. Data We Do Not Collect

  • Names, emails, phone numbers, or contact information
  • Photos, files, or documents
  • Clipboard content
  • SMS or call logs
  • Precise location
  • Biometric data beyond what Health Connect optionally provides on-device
  • Advertising identifiers used for targeted advertising

5. Data Retention and Deletion

  • Local data (app layout, time limits, usage history): retained until the app is uninstalled or the user clears app data.
  • Analytics data: retained by Firebase per their data retention settings.
  • To delete all local data: uninstall the App or go to Android Settings → Apps → Fokus → Clear Data.

6. Children's Privacy

Fokus is not directed to children under the age of 13. We do not knowingly collect personal information from children.
